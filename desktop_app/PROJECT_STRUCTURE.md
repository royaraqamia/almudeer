```bash
# macOS
flutter build macos --release
```

### 3. Code Migration Strategy
You have two options:

**Option A: Share Code with Mobile App (Recommended)**
- Create a shared `common/` directory at the root level
- Move shared code (services, models, utilities) there
- Both mobile-app and desktop-app import from common

**Option B: Copy Relevant Code**
- Copy needed features from mobile-app/lib/features
- Adapt platform-specific code for desktop
- Remove mobile-only dependencies

### 4. Platform-Specific Considerations

**Remove/Conditional:**
- Firebase messaging (use desktop notifications instead)
- Mobile-specific sensors (proximity, battery)
- CallKit (iOS only)
- Android intents
- WorkManager (use desktop background tasks)

**Add for Desktop:**
- File system access (native file picker)
- Window management
- System tray integration
- Keyboard shortcuts
- Print support (if needed)

### 5. Backend Integration
The desktop app needs to connect to your existing backend at:
- Backend URL (configure in .env or settings)
- Same API endpoints as mobile app
- WebSocket support for real-time features

### Linux

- Creates executable bundle
- Can package as Snap or AppImage

### macOS
Output: `desktop_app\build\macos\Build\Products\Release\`

## Key Differences from Mobile App

1. **No Splash Screen** - Desktop apps launch differently
2. **Window Management** - Need to handle resize, minimize, maximize
3. **File System** - Direct access to user's file system
4. **Notifications** - Use desktop notification system
5. **No Mobile Sensors** - Remove proximity, battery optimization
6. **Keyboard Navigation** - Desktop users expect keyboard shortcuts

## Dependencies to Remove from Mobile Version

These mobile-specific packages won't work on desktop:
- `firebase_messaging` (use desktop notifications)
- `firebase_crashlytics` (optional, can keep for web)
- `flutter_callkit_incoming` (iOS only)
- `android_intent_plus` (Android only)
- `workmanager` (mobile only)
- `receive_sharing_intent` (mobile only)
- `nearby_connections` (mobile only)
- `proximity_sensor` (mobile only)
- `battery_plus` (optional on desktop)
- `just_waveform` (may have desktop issues)

## Alternative Desktop-Safe Packages

Replace with:
- `desktop_notifications` - For system notifications
- `file_selector` - Native file picker
- `bitsdojo_window` - Alternative window management
- `hotkey_manager` - Keyboard shortcuts
- `printer` - Print support (if needed)
